Update Code Wiki content for changed modules.

Argument: $ARGUMENTS (optional: module path or "full" for complete rebuild)

IMPORTANT: You MUST use the `wiki_update` MCP tool (mcp__code-wiki__wiki_update). Do NOT fall back to `npx code-wiki update` or any Bash/CLI command. The two-phase MCP workflow produces higher quality content than the CLI.

Steps:
1. If a module path is provided, call `wiki_update({ scope: "changed", paths: ["<module_path>"] })`
   If "full" is specified, call `wiki_update({ scope: "full" })`
   Otherwise, call `wiki_update({ scope: "changed" })`
2. Review the analysis data returned (AST summary + dependency graph)
3. Generate wiki content from the analysis: module descriptions, gotchas, boundary definitions
4. Call `wiki_update({ module: "<name>", generated_content: "<json_string>" })` to persist each module's wiki
5. After all modules are persisted, generate and persist the project overview:
   Call `wiki_update({ persist_overview: "<json_string>" })`
6. Optionally generate tech stack descriptions:
   Call `wiki_update({ persist_tech_stack_descriptions: "<json_string>" })`
7. Report what was updated

The two-step pattern ensures wiki content is generated by the host AI (you) without extra API cost.
